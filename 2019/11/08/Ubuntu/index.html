Ubuntu速学笔记

发表于 2019-11-08 | 分类于 拾级而上 ， Linux ， Ubuntu | 字数统计: 1k 字 | 阅读时长 ≈ 4分钟

为了学习docker来回顾Linux基础，本篇是基于Ubuntu操作系统安装部署遇到问题或有用知识点的汇总整理。

ifconfig命令找不到

问题描述： 在Vmware安装完成Ubuntu后，输入ifconfig提示：Command ‘ifconfig’ not found, but can be installed with: sudo apt install net-tools。当然官方提醒我们可以使用sudo apt install net-tools安装net-tools，然而并不如预想。

解决方案： 执行语句 ` sudo apt-get update && sudo apt-get install net-tools` 再输入ifconfig就正常啦。(apt-get用于自动从互联网的软件仓库中搜索、安装、升级或卸载软件。我们需要顶起执行`apt-get update`，让我们缓存在系统中的软件列表源是最新的)

如果在过程中提示 E: Could not get lock /var/lib/apt/lists/lock - open (11: Resource temporarily unavailable)， 不用担心，这是因为上一次安装时异常退出导致锁残留，可直接删除掉这个锁文件，使用命令sudo rm /var/lib/apt/lists/lock。

编译安装PHP

  1. 官网下载安装包到本地

  2. 执行预编译，使用./configure --prefix=/usr/local/php(这个后面最好加上--with-config-file-path=/usr/local/php/conf，代表指定php.ini路径，否则后面需要自己去设置) 代表我要装到/usr/local/php里，配置文件放到conf里

    a. 如果遇到错误：error:xml2-config not found.Please check your libxml2 installation. ，说明依赖包libxml2不存在，需要安装使用命令apt-get install libxml2-dev -y。
    b. 如果安装libxml2过程又提示Unable to fetch some archives, maybe run apt-get update or try with –fix-missing?，可执行apt-get install libxml2-dev -y --fix-missing解决。
    c. 如果又报错configure: error: no acceptable C compiler found in $PATH，是因为你本地没有可执行编译的源文件，需要执行sudo apt-get update && sudo apt-get install gcc -y解决。

  3. 安装，执行完成之后最后一定要执行make，这一步是真正的编译。然后还要执行sudo make install，这一步是拷贝到你的目标文件夹，否则/usr/local/php是不会有的。

  4. 验证，没有Hello world感觉不完整是不是。写个简单的PHP页面

环境变量

基于用户级的环境变量设置 在home目录下的隐藏文件 .bashrc，输入export name=xugz设置个名字，使用source .bashrc执行。比如说我现在安装了Nodejs，但是只能在node安装目录下进行操作，如何处理呢？可以通过修改环境变量实现。在.bashrc文件中追加export PATH=$PATH:/home/xugz/tool/node-v12.13.0-linux-x64/bin后保存退出，使用source .bashrc执行就可以在任何文件目录下执行node操作了。

Mysql8.0.18源码安装

  1. 官网下载源码 社区版 Compressed TAR Archive, Includes Boost Headers
  2. 安装依赖编译工具
    a. CMake(CMake是一个跨平台的安装（编译）工具，可以用简单的语句来描述所有平台的安装（编译过程）。他能够输出各种各样的makefile)。我这里直接使用sudo apt-get update && sudo apt-get install cmake -y执行安装。如果版本过低，可以去https://cmake.org/files/v3.4/cmake-3.4.1.tar.gz下载后手动编译安装。
    b. GIT 安装 使用 sudo apt-get update && sudo apt-get install git -y
    c. c/c++编译器 使用 sudo apt-get update && sudo apt-get install gcc g++ -y
    d. Linux常用图形库 使用 sudo apt-get update && sudo apt-get install libncurses5 libncurses5-dev -y
    e. 还有些其他的 sudo apt install build-essential bison libssl-dev pkg-config
  3. 新建 /usr/local/mysql文件, 修改归属用户权限
  4. 编译 执行cmake . -DCMAKE_INSTALL_PREFIX=/usr/local/mysql -DSYSCONFDIR=/etc -DDEFAULT_CHARSET=utf8 -DDEFAULT_COLLATION=utf8_general_ci -DMYSQL_DATADIR=/var/lib/mysql/data -DWITH_BOOST=boost，如果没有草错，执行make编译，然后开始漫长的等待，漫长的等待。完成后执行 make install 执行安装
